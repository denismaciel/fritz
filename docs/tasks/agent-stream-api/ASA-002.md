# ASA-002: define internal event model and run lifecycle ids

**Type:** Task

## Goal

Define one protocol-neutral event model for agent runs, with stable run ids and event ordering.

## Places To Change

- `internal/agent/` or equivalent new package
- `internal/app/`
- `internal/chat/`
- `internal/session/`
- `internal/tool/`

## Scope

- define typed events for run started, text delta, message completed, tool started, tool completed, step started, step completed, error, canceled, run finished
- add run ids and event ids/correlation ids where needed
- emit these events from the headless service
- add focused tests for event ordering in common flows

## Non-goals

- wire protocol-specific structs
- expose HTTP yet
- implement AI SDK event names

## Design Notes

- internal events are the source of truth; AG-UI and AI SDK are adapters
- event payloads should be typed, not `map[string]any`
- keep text streaming and tool execution represented distinctly
- include enough metadata for clients to correlate tool calls and final messages

## Acceptance Criteria

- service emits typed events for plain text runs and tool-using runs
- run ids are stable across all events of one run
- tests pin ordering for at least text, tool, error flows

## Dependencies

- `ASA-001`

## Validation

- `go test ./...`
- `rg -n "RunStarted|TextDelta|ToolStarted|RunFinished" internal`
