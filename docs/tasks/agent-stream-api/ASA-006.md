# ASA-006: add thin CLI client over AG-UI SSE

**Type:** Task

## Goal

Add a simple client mode that talks to the HTTP AG-UI server instead of embedding the runtime.

## Places To Change

- `cmd/fritz/`
- `internal/app/`
- `internal/httpapi/` or equivalent
- `internal/command/`

## Scope

- add a client command or flags for remote server mode
- consume AG-UI SSE from HTTP
- render text deltas, tool notices, final messages, and errors in plain terminal output
- keep UX intentionally simple; line-oriented is enough

## Non-goals

- full-screen TUI
- rich event visualization
- browser code

## Design Notes

- client should parse AG-UI, not internal event structs
- keep formatting logic local to client code
- support same plain command set at minimum for one-shot run; interactive chat can be partial if clearly bounded

## Acceptance Criteria

- CLI can target local server and display streamed agent output
- client handles tool events and errors cleanly
- tests cover parsing/rendering of representative AG-UI SSE streams

## Dependencies

- `ASA-005`

## Validation

- `go test ./...`
- `go run ./cmd/fritz --help`
