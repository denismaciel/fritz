# ASA-005: add HTTP server endpoints for runs and cancel

**Type:** Task

## Goal

Expose the headless service over HTTP with AG-UI SSE streaming and cancel endpoints.

## Places To Change

- `cmd/fritz/`
- `internal/httpapi/` or equivalent new package
- `internal/agent/`
- `internal/protocol/agui/`
- `internal/command/`
- `internal/config/`

## Scope

- add server command, eg `fritz serve`
- implement `POST /runs` returning `text/event-stream`
- implement `POST /runs/{id}/cancel`
- support request decoding from JSON into service run requests
- add config/flags for bind address

## Non-goals

- authentication
- browser app
- WebSocket transport
- persisted run replay

## Design Notes

- keep HTTP handler layer thin
- do not let handlers know tool/model internals
- request model should be explicit and versionable
- return plain JSON errors for non-stream endpoints and early request failures

## Acceptance Criteria

- a local HTTP server can stream a run over SSE
- cancel endpoint stops an active run by id
- existing CLI commands still work

## Dependencies

- `ASA-001`
- `ASA-003`
- `ASA-004`

## Validation

- `go test ./...`
- `go run ./cmd/fritz serve --help`
