# ASA-007: add AI SDK UI message stream adapter

**Type:** Task

## Goal

Expose the same service runs through an AI SDK UI Message Stream-compatible endpoint.

## Places To Change

- `internal/protocol/aisdk/` or equivalent new package
- `internal/httpapi/`
- `internal/agent/`
- tests under `internal/protocol/` or equivalent

## Scope

- map internal events to AI SDK UI Message Stream parts
- add a separate HTTP endpoint, eg `POST /ai-sdk/chat`
- support text, tool input/output, finish, error, and custom data parts where needed
- keep adapter independent from AG-UI encoder

## Non-goals

- replacing AG-UI as canonical protocol
- React component code
- legacy ad hoc stream formats

## Design Notes

- this is an adapter, not the core event model
- preserve internal event ordering
- use custom data parts only where there is no native AI SDK part
- keep endpoint request/response shape documented and narrow

## AI SDK reference docs

- `https://ai-sdk.dev/docs/ai-sdk-ui/stream-protocol`
- `https://ai-sdk.dev/docs/reference/ai-sdk-ui/read-ui-message-stream`

## Acceptance Criteria

- endpoint emits an AI SDK UI-compatible stream for representative runs
- tests pin key part types and finish semantics
- adapter stays decoupled from AG-UI package internals

- `ASA-002`
- `ASA-005`
