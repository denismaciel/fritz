# ASA-001: extract headless agent service boundary

**Type:** Task

## Goal

Extract a reusable in-process `AgentService` from `internal/app` so frontends can drive runs without owning stdin/stdout or protocol code.

## Places To Change

- `internal/app/`
- `internal/chat/`
- `internal/session/`
- `internal/prompt/`
- `internal/model/`
- `cmd/fritz/`

## Scope

- define a typed service boundary for starting a run against the existing runtime
- move orchestration now buried in `internal/app/app.go` behind service methods
- keep current CLI commands working by calling the new service
- define typed request structs for one-shot run and interactive run setup

## Non-goals

- HTTP server
- SSE encoding
- browser client code
- AI SDK adapter

## Design Notes

- keep the new service unaware of concrete transports
- avoid moving provider/tool/session internals unless required by the seam
- current `app.Run` should become mostly command resolution plus frontend wiring
- service output should not be raw text; later tickets need typed events

## Acceptance Criteria

- a new service package or service type exists and can be invoked from tests without stdin/stdout
- `cmd/fritz` still supports current `run`, `chat`, `doctor`, `help`
- orchestration logic is thinner in `internal/app/app.go`

## Dependencies

- `None.`

## Validation

- `go test ./...`
- `rg -n "type .*Service|StartRun|RunRequest" internal`
