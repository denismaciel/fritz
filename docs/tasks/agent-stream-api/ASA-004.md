# ASA-004: implement AG-UI SSE encoder

**Type:** Task

## Goal

Encode the internal event stream as AG-UI-compatible SSE events.

## Places To Change

- `internal/protocol/agui/` or equivalent new package
- `internal/agent/`
- `internal/model/`
- tests under `internal/protocol/` or equivalent

## Scope

- map internal events to AG-UI event names and payloads
- implement `text/event-stream` framing
- keep encoder decoupled from HTTP handlers so it can be tested as bytes/records
- include event shapes for text, tool, lifecycle, error, cancel

## Non-goals

- actual HTTP endpoint
- AI SDK adapter
- browser client code

## Design Notes

- follow AG-UI docs current as of this pack
- isolate protocol structs under one package
- preserve ordering from internal events; encoder should not reorder
- if internal events do not map 1:1, keep adapter logic local here

## AG-UI reference docs

- `https://docs.ag-ui.com/introduction`
- `https://docs.ag-ui.com/concepts/architecture`
- `https://tanstack.com/ai/latest/docs/protocol/chunk-definitions`

## Acceptance Criteria

- encoder emits valid SSE records for representative event sequences
- tests pin event names and core payload fields
- no AG-UI types leak upward into service/core packages

## Dependencies

- `ASA-002`

## Validation

- `go test ./...`
- `rg -n "text/event-stream|AG-UI|agui" internal`
