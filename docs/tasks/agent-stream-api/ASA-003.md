# ASA-003: add cancellation and run registry

**Type:** Task

## Goal

Support long-lived concurrent runs with explicit cancellation and lookup by run id.

## Places To Change

- `internal/agent/` or equivalent new package
- `internal/app/`
- `internal/session/`
- `internal/tool/bash.go`
- `internal/gemini/`

## Scope

- add a run registry owned by the service
- support cancel by run id
- propagate cancellation through model calls and long-running tools
- expose typed terminal states for canceled vs failed

## Non-goals

- resumable event replay
- persistent queueing
- multi-process coordination

## Design Notes

- cancellation should be context-driven end to end
- registry only needs in-memory lifecycle tracking for now
- cleanup must remove completed/canceled runs to avoid leaks
- do not add speculative pause/resume semantics

## Acceptance Criteria

- active runs can be canceled by id
- canceled runs emit canceled/finished states predictably
- tests cover cancel during model work and cancel during tool work

## Dependencies

- `ASA-002`

## Validation

- `go test ./...`
- `rg -n "Cancel|Registry|RunStateCanceled" internal`
