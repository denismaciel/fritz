# GHB-004: Telegram delivery integration

**Type:** Task

## Goal

Wire heartbeat into the Telegram-facing gateway flow so actionable results get delivered once.

## Places To Change

- `internal/heartbeat/`
- `internal/telegram/`
- `internal/app/`

## Scope

- integrate scheduler + queue with Telegram-capable gateway runtime
- deliver actionable heartbeat output back to Telegram
- suppress no-op heartbeat output
- tests for one delivered message and deduped wakes

## Non-goals

- webhook support
- rich formatting
- admin/operator commands

## Design Notes

- Telegram adapter should stay transport-thin
- use existing send path rather than inventing a second delivery path
- do not allow overlapping sends for same wake target

## Acceptance Criteria

- heartbeat can produce a Telegram outbound message
- no-op heartbeat produces nothing
- duplicate queued wakes do not spam users

## Dependencies

- `GHB-003`

## Validation

- `go test ./internal/telegram ./internal/app -run 'Test.*Heartbeat|TestRunTelegram'`
- `go test ./internal/heartbeat`
