# GHB-003: heartbeat engine invocation contract

**Type:** Task

## Goal

Define how heartbeat invokes the engine and how no-op vs actionable output is interpreted.

## Places To Change

- `internal/heartbeat/`
- `internal/engine/`
- `internal/gateway/`

## Scope

- define heartbeat prompt contract
- add one explicit no-op sentinel such as `HEARTBEAT_OK`
- invoke engine through existing engine seam
- map actionable output vs no-op result
- tests for no-op suppression and actionable run results

## Non-goals

- Telegram delivery
- reminder scheduling
- multiple prompt modes

## Design Notes

- keep contract explicit and boring
- engine should receive a normal prompt; heartbeat package decides interpretation
- no-op detection must be deterministic

## Acceptance Criteria

- heartbeat can invoke engine for one wake target
- no-op result is suppressed
- actionable result is returned to caller

## Dependencies

- `GHB-001`
- `GHB-002`

## Validation

- `go test ./internal/heartbeat -run 'Test.*Engine|Test.*NoOp'`
- `rg -n "HEARTBEAT" internal/heartbeat internal/gateway`
