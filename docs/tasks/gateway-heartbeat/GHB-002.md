# GHB-002: heartbeat scheduler + persistence

**Type:** Task

## Goal

Add a simple scheduler loop and persist minimal heartbeat state under gateway state.

## Places To Change

- `internal/heartbeat/`
- `internal/gateway/`

## Scope

- simple interval scheduler
- persist heartbeat metadata under `.fritz/gateway/`
- store enough state for restart continuity:
  - last tick
  - last successful run
  - pending wakes if needed
- tests for scheduler tick behavior and state persistence

## Non-goals

- rich cron expressions
- retries/backoff policy
- multiple work sources

## Design Notes

- keep persistence small and inspectable
- one JSON file is fine for heartbeat state
- scheduler should be stop/start safe

## Acceptance Criteria

- heartbeat scheduler can tick on interval
- heartbeat state persists across restart
- tests cover persisted state round-trip

## Dependencies

- `GHB-001`

## Validation

- `go test ./internal/heartbeat -run 'Test.*Scheduler|Test.*State'`
- `rg -n "heartbeat" internal/heartbeat internal/gateway`
