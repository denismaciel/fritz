# GHB-001: heartbeat domain model + wake queue

**Type:** Task

## Goal

Create the pure heartbeat domain types and a coalescing wake queue.

## Places To Change

- `internal/heartbeat/`
- `internal/gateway/`

## Scope

- define wake target identity and wake reason types
- define enqueue/dequeue API
- coalesce duplicate wakes by target key
- add pure tests for queue semantics

## Non-goals

- timers
- engine invocation
- Telegram delivery

## Design Notes

- target key should line up with gateway session routing
- queue should be generic enough for future reminders/tasks
- this is the core seam everything else depends on

## Acceptance Criteria

- queue dedupes duplicate wakes for same target
- queue preserves pending work for distinct targets
- tests cover enqueue/dequeue/coalesce behavior

## Dependencies

- `None.`

## Validation

- `go test ./internal/heartbeat -run Test`
- `rg -n "Wake|Queue|Coalesce" internal/heartbeat`
