# TRU-004 discovery tool hardening

## Goal

Improve `grep` / `find` / `ls` semantics for real repos, especially ignore handling, glob semantics, and scale.

## Places To Change

- `internal/tool/discovery.go`
- `internal/tool/common.go`
- discovery tests under `internal/tool/`

## Scope

- improve `.gitignore` handling
- improve glob semantics
- consider shelling out to `rg` / `fd` when available on Unix
- keep same tool names and current core contract

## Non-goals

- new discovery tools
- Windows support
- UI changes

## Design Notes

- behavior correctness matters more than perfect purity here
- shelling out is acceptable if wrapped cleanly and testably
- if shelling out is chosen, keep a fallback or explicit error path

## Acceptance Criteria

- discovery tools behave better on nested repos and ignored files
- tests cover nested ignore cases and glob edge cases
- current parity validator still passes

## Dependencies

- `TRU-001`

## Validation

- `go test ./...`
- `uv run scripts/validate_pi_tool_test_parity.py`
