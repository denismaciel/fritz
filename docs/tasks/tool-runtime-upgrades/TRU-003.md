# TRU-003 unix bash runtime upgrade

## Goal

Upgrade bash runtime on Unix to support partial output, better cancellation/timeout behavior, and bounded output handling.

## Places To Change

- `internal/tool/bash.go`
- `internal/app/app.go`
- `internal/model/model.go` if streaming event shape must widen
- tests under `internal/tool/` and `internal/app/`

## Scope

- stream partial bash output
- preserve timeout semantics
- make cancellation actually interrupt command execution
- cap in-memory output, with spill-to-temp-file if needed
- surface typed bash details for truncation/full-output-path

## Non-goals

- Windows support
- interactive shell sessions
- full TUI parity

## Design Notes

- keep execution backend behind `BashOperations`
- Unix process cleanup matters; handle child termination pragmatically
- use structured tool payloads from `TRU-001`

## Acceptance Criteria

- long-running commands can emit partial output
- timeout/cancel tests cover actual process interruption
- large output no longer requires full buffering in memory
- current parity tests still pass

## Dependencies

- `TRU-001`
- `TRU-002`

## Validation

- `go test ./...`
- `uv run scripts/validate_pi_tool_test_parity.py`
