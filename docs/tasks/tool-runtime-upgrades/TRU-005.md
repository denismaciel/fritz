# TRU-005 injectable fs ops seams

## Goal

Add injectable operation seams for filesystem-backed tools so they stay testable and can later target remote/containerized backends.

## Places To Change

- `internal/tool/read.go`
- `internal/tool/write.go`
- `internal/tool/edit.go`
- `internal/tool/discovery.go`
- related constructors and tests

## Scope

- define ops interfaces for read/write/edit/discovery where missing
- thread those seams through constructors/options
- update tests to use seams where it improves clarity

## Non-goals

- implementing remote backends
- changing CLI behavior
- adding extension runtime

## Design Notes

- keep default local filesystem path boring
- the seam should isolate fs/process details, not leak them into chat/model code
- do not reintroduce generic maps while doing this

## Acceptance Criteria

- fs-backed tools can be instantiated with injected ops
- tests can cover core behavior without always touching real fs/processes
- current parity validator still passes

## Dependencies

- `TRU-001`
- `TRU-004`

## Validation

- `go test ./...`
- `uv run scripts/validate_pi_tool_test_parity.py`
