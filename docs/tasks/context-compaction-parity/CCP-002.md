# CCP-002: local compaction prompt templates + structured summary contract

**Type:** Task

## Goal

Replace ad hoc compaction prompt construction with dedicated local prompt templates and a stable structured summary contract aligned with Codex concepts.

## Places To Change

- `internal/session/compaction.go`
- `internal/prompt/`
- `internal/prompt/resources.go`
- `internal/prompt/resources_test.go`
- `internal/session/manager_test.go`

## Scope

- move compaction prompt text into dedicated local template/resource files
- define the summary contract fields:
  - goal
  - constraints
  - progress
  - key findings
  - next steps
- keep a distinct handoff prefix / framing step if needed by local message model
- add tests for prompt loading and summary formatting expectations

## Non-goals

- token-threshold trigger logic
- checkpoint persistence schema
- mid-turn compaction behavior

## Design Notes

- use the extracted Codex refs as comparison material, not direct runtime dependencies
- keep prompt loading deterministic and easy to diff in tests
- preserve ability to pass custom instructions on top of the base template
- summaries should optimize for post-compact continuation, not human prose

## Acceptance Criteria

- local compaction prompt content is no longer hardcoded inline in `summarizeTurns`
- tests assert template loading and summary contract shape
- compaction summaries preserve clear next-step data in tests

## Dependencies

- `CCP-001`

## Validation

- `go test ./internal/session ./internal/prompt`
- `rg -n "Summarize this fritz session|goal|next step" internal/session internal/prompt`
