# CCP-007: mid-turn compaction + overflow fallback cleanup

**Type:** Task

## Goal

Support compaction between tool-result accumulation and the next model step, and demote overflow retry to true fallback behavior.

## Places To Change

- `internal/agent/service.go`
- `internal/chat/core.go`
- `internal/session/compaction.go`
- `internal/agent/service_test.go`

## Scope

- estimate follow-up request size after tool outputs
- compact before the follow-up model call when above threshold
- keep user goal / pending prompt intact across mid-turn compaction
- simplify overflow retry path so it only handles estimator misses or hard edge cases
- add tests for tool-heavy mid-turn compaction

## Non-goals

- trimming oversized compaction inputs
- remote compact API parity
- realtime-specific behavior

## Design Notes

- preserve current tool loop semantics; compaction should be inserted, not redesign the loop
- avoid losing pending prompt state during compaction
- keep logs explicit enough to tell pre-turn vs mid-turn compaction apart

## Acceptance Criteria

- tool-heavy turns can compact and continue within the same task
- overflow retry remains available but is no longer the primary compaction entrypoint
- local tests cover mid-turn compaction and post-tool continuation
- Codex parity validator reflects mirrored mid-turn compaction names where applicable

## Dependencies

- `CCP-004`
- `CCP-006`

## Validation

- `go test ./internal/agent ./internal/chat ./internal/session`
- `uv run scripts/validate_codex_compaction_test_parity.py`
