# CCP-006: checkpoint-based context rebuild + resume/fork parity

**Type:** Task

## Goal

Make session context rebuild start from the latest compaction checkpoint and replay later events so resume/fork behavior matches the compacted model view.

## Places To Change

- `internal/session/manager.go`
- `internal/session/runtime.go`
- `internal/session/manager_test.go`

## Scope

- change `BuildContext()` to seed from the latest checkpoint when present
- replay later messages/events on top of that checkpoint
- preserve branch summary behavior
- add tests for:
  - resume after compaction
  - repeated compactions
  - fork after compaction

## Non-goals

- rollback semantics beyond what current session model supports
- mid-turn compaction
- low-value trimming

## Design Notes

- this is the main architectural parity step
- keep replay logic deterministic and isolated
- prefer one reconstruction path for fresh and resumed sessions
- mirror Codex resume/fork test names where local concepts map cleanly

## Acceptance Criteria

- resumed sessions rebuild from latest checkpoint instead of loose summary-only state
- forked sessions preserve compacted history view
- repeated compactions keep the latest compacted base correct
- local parity validator reflects new resume/fork mirrored tests

## Dependencies

- `CCP-005`

## Validation

- `go test ./internal/session`
- `uv run scripts/validate_codex_compaction_test_parity.py`
