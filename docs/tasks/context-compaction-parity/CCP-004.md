# CCP-004: pre-turn proactive compaction

**Type:** Task

## Goal

Compact before the next model request when estimated request size crosses the configured threshold.

## Places To Change

- `internal/agent/service.go`
- `internal/session/compaction.go`
- `internal/session/manager_test.go`
- `internal/agent/service_test.go`

## Scope

- estimate request size before each `Generate`
- run compaction before the model call when over threshold
- keep existing overflow retry path as fallback
- update state after proactive compaction so the next request uses compacted context
- add tests covering proactive pre-turn compaction

## Non-goals

- mid-turn compaction after tool outputs
- checkpoint persistence
- low-value history trimming

## Design Notes

- this ticket should be the first user-visible behavior win
- keep the trigger path explicit and easy to log/debug
- avoid silently compacting multiple times in one pre-turn phase
- update local tests to mirror Codex pre-turn compact cases by normalized name where feasible

## Acceptance Criteria

- model calls compact first when prompt estimate exceeds threshold
- overflow retry still works when estimation misses
- local tests cover pre-turn compact trigger and non-trigger cases
- Codex parity validator shows mirrored names for the adopted pre-turn cases

## Dependencies

- `CCP-002`
- `CCP-003`

## Validation

- `go test ./internal/agent ./internal/session`
- `uv run scripts/validate_codex_compaction_test_parity.py`
