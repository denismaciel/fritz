# CCP-001: Codex parity validator + local compaction inventory

**Type:** Task

## Goal

Turn the extracted Codex compaction test inventory into a usable local parity validator and a local compaction test inventory so pack progress is measured mechanically.

## Places To Change

- `scripts/extract_codex_compaction_tests.py`
- `scripts/extract_codex_compaction_tests_test.py`
- `scripts/validate_codex_compaction_test_parity.py`
- `scripts/validate_codex_compaction_test_parity_test.py`
- `docs/reference/codex/compaction-tests.json`
- `internal/session/`
- `internal/agent/`

## Scope

- add a local Go compaction test extractor for `Test*` and `t.Run(...)`
- add a validator that compares normalized Codex compaction test names vs local Go compaction tests
- scope local extraction to compaction-related packages/tests only
- print missing local parity tests with closest matches
- make the validator fail when Codex compaction tests are not mirrored locally
- document the validator command in pack notes if paths change

## Non-goals

- implementing compaction behavior changes yet
- perfect semantic equivalence checking
- CI wiring outside this repo unless needed for validation docs

## Design Notes

- follow the style of `scripts/validate_pi_tool_test_parity.py`
- normalized test-name parity is the guardrail; keep normalization simple and stable
- do not hand-maintain Codex test names in markdown
- local inventory should include only compaction-relevant Go tests to avoid noisy diffs

## Acceptance Criteria

- `uv run scripts/validate_codex_compaction_test_parity.py` runs and reports missing local mirrors
- script tests cover Rust extraction, Go extraction, normalization, and missing-test reporting
- validator is usable as a required validation step for later tickets

## Dependencies

- `None.`

- `uv run scripts/validate_codex_compaction_test_parity_test.py`
- `uv run scripts/validate_codex_compaction_test_parity.py`
