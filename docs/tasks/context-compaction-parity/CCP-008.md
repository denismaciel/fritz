# CCP-008: compaction fit trimming + repeated-compaction hardening

**Type:** Task

## Goal

Make compaction itself resilient when history is already oversized, and harden repeated-compaction behavior against pathological long sessions.

## Places To Change

- `internal/session/compaction.go`
- `internal/session/manager.go`
- `internal/agent/service.go`
- `internal/session/manager_test.go`
- `internal/agent/service_test.go`

## Scope

- trim low-value old items when the compaction request itself would exceed budget
- bias trimming toward bulky old tool outputs before recent user intent
- cover repeated compactions and mixed pre-turn/mid-turn sequences
- add or update mirrored hardening tests from Codex inventory

## Non-goals

- exact Codex normalization of ghost snapshots or realtime artifacts
- large architectural refactors unrelated to compaction

## Design Notes

- keep trimming heuristics explicit and conservative
- prefer dropping old bulky tool payloads over recent user-visible turns
- do not let trimming silently remove the current task goal
- this ticket should close the largest remaining parity gaps after the main behavior lands

## Acceptance Criteria

- compaction can still run when history is too large for the first compaction attempt
- repeated compactions preserve the latest relevant user history
- local tests cover oversized-compaction-input and repeated-compaction cases
- Codex parity validator is green for the full adopted compaction scope
