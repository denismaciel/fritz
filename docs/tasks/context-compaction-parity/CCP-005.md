# CCP-005: compaction checkpoint persistence schema

**Type:** Task

## Goal

Upgrade compaction persistence from summary-only metadata to a real checkpoint that can seed later context reconstruction.

## Places To Change

- `internal/session/manager.go`
- `internal/session/compaction.go`
- `internal/session/manager_test.go`

## Scope

- extend compaction entries to persist replacement messages/history
- preserve existing summary fields where still useful
- keep session file format backward-compatible with older summary-only entries
- add tests for writing and reading compaction checkpoints

## Non-goals

- using the checkpoint as the primary rebuild path yet
- mid-turn compaction
- migration tooling beyond lazy backward compatibility

## Design Notes

- treat the checkpoint as the new compacted history base
- do not persist duplicate data unless it materially simplifies rebuild logic
- keep JSONL entries readable and version-tolerant
- prefer additive schema changes

## Acceptance Criteria

- compaction entries can persist replacement history/messages
- old session files still open
- tests cover checkpoint append and reload behavior

## Dependencies

- `CCP-002`
- `CCP-003`

## Validation

- `go test ./internal/session`
- `rg -n "Replacement|CompactionEntryType|FirstKeptEntryID" internal/session`
