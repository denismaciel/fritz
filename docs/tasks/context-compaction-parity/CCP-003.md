# CCP-003: token accounting + compaction config thresholds

**Type:** Task

## Goal

Add token-oriented compaction accounting and config so compaction decisions use request size, not only turn count.

## Places To Change

- `internal/model/model.go`
- `internal/config/config.go`
- `internal/session/manager.go`
- `internal/session/runtime.go`
- `internal/session/planning.go`
- `internal/session/planning_test.go`

## Scope

- add rough token estimator for system prompt, messages, and tool definitions
- add config for token thresholds and optional compaction target
- store enough token state to distinguish:
  - last known prompt size estimate
  - current threshold values
- keep existing turn-based settings temporarily as fallback or compatibility knobs
- add tests for threshold planning behavior

## Non-goals

- triggering compaction in the run loop yet
- checkpoint persistence
- trimming oversized compaction inputs

## Design Notes

- approximate byte-based estimation is acceptable initially
- keep token accounting cheap and deterministic
- avoid threading token logic through unrelated packages
- prefer one small estimator API over scattered helper math

## Acceptance Criteria

- local config supports token-based compaction thresholds
- planning tests cover below-threshold and above-threshold cases
- code can estimate compaction pressure without calling the model

## Dependencies

- `CCP-001`

## Validation

- `go test ./internal/model ./internal/config ./internal/session`
- `rg -n "CompactThresholdTokens|Estimate.*Token" internal/model internal/config internal/session`
