# PLS-002: skill loader and prompt formatting core

**Type:** Task

## Goal

Implement Pi-shaped skill loading: frontmatter parsing, discovery from configured roots, diagnostics, and prompt-safe skill summary formatting.

## Places To Change

- `internal/skill/`
- `internal/prompt/`
- `go.mod`

## Scope

- define skill type with name, description, file path, base dir, and disable-model-invocation flag
- load skills from explicit roots and discovered default roots
- parse frontmatter and keep warnings separate from fatal errors
- format visible skills into prompt text
- dedupe/collision behavior for duplicate names

## Non-goals

- automatic model-chosen skill body loading
- `/skill:name` expansion
- prompt templates
- package-manager style discovery

## Design Notes

- keep loader separate from prompt builder
- keep diagnostics typed; do not print directly from core
- first pass can support project/global roots plus explicit paths; no Pi package discovery needed
- prompt summary should include enough for progressive disclosure, not full skill bodies

## Acceptance Criteria

- skills load from configured roots with stable ordering
- malformed skills yield diagnostics, not panics
- `disable-model-invocation` skills are excluded from prompt summary
- focused unit tests cover parsing, discovery, collisions, and formatting
