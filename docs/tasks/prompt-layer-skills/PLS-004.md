# PLS-004: skill invocation expansion

**Type:** Task

## Goal

Support explicit `/skill:name` invocation by expanding the referenced `SKILL.md` into the conversation in a controlled, typed way.

## Places To Change

- `internal/skill/`
- `internal/chat/`
- `internal/app/`
- `internal/model/`

## Scope

- detect `/skill:name` user input
- resolve named skill from loaded registry
- load full skill body on demand
- expand to a structured block that preserves skill name, source path, and optional user args
- keep relative-path guidance in the expanded content

## Non-goals

- automatic skill selection by the model
- prompt templates
- extension slash commands
- interactive autocomplete or TUI affordances

## Design Notes

- preserve progressive disclosure: summary in system prompt, full body only on explicit invocation
- expanded format should be stable and testable, not ad hoc string glue spread across app code
- unknown skill names should fail clearly
- keep expansion before model call, outside provider code

## Acceptance Criteria

- `/skill:name` expands to full skill content plus args
- unknown skill names return a clear error
- expanded content includes path/base-dir guidance
- tests cover invocation, args passing, and failure cases
