# PLS-001: context and prompt resource discovery

**Type:** Task

## Goal

Add typed discovery for repo/global prompt-layer resources: `AGENTS.md` or `CLAUDE.md`, `SYSTEM.md`, `APPEND_SYSTEM.md`, and skill roots.

## Places To Change

- `internal/config/`
- `internal/app/`
- `internal/prompt/` or equivalent new package

## Scope

- discover global and project prompt resources
- walk parent dirs for `AGENTS.md` / `CLAUDE.md`
- discover `.fritz` prompt files for project-local overrides
- define typed result structs for discovered resources
- make discovery testable without model/network code

## Non-goals

- system prompt string assembly
- skill markdown parsing
- `/skill:name` expansion
- prompt templates

## Design Notes

- prefer a dedicated prompt/resource package over sprinkling file reads through app code
- keep discovery output typed and provider-agnostic
- preserve deterministic ordering: global first, then ancestors, then cwd
- day-one local convention can be `.fritz/SYSTEM.md` and `.fritz/APPEND_SYSTEM.md`

- missing files are handled cleanly
- discovery is covered by focused tests
