# PLS-003: system prompt builder and request boundary

**Type:** Task

## Goal

Add a real system prompt builder and thread it into the model request boundary so prompt-layer state is no longer implicit app glue.

## Places To Change

- `internal/prompt/`
- `internal/model/model.go`
- `internal/chat/core.go`
- `internal/app/app.go`
- `internal/gemini/client.go`

## Scope

- define typed prompt-builder inputs: tool snippets, guidelines, context files, skills, cwd, date, overrides
- add system prompt field to model request shape
- compose system prompt from discovered resources and active tool metadata
- send system prompt through the Gemini boundary cleanly
- keep prompt building deterministic and unit-testable

## Non-goals

- prompt templates
- skill body expansion
- extension overrides
- provider-specific prompt behavior beyond Gemini mapping

## Design Notes

- do not bury prompt construction inside app orchestration
- system prompt should rebuild from current runtime/tool state, not static startup text only
- preserve clean boundary: app resolves resources, prompt package builds, model gateway just transports
- keep tool-derived prompt text separate from raw tool defs
