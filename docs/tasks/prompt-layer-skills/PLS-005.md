# PLS-005: CLI and config seams for skills and prompt files

**Type:** Task

## Goal

Expose enough config surface to make skills and prompt-layer files usable without hardcoding paths.

## Places To Change

- `internal/config/`
- `internal/command/`
- `internal/app/`
- `README.md`

## Scope

- add config/file/env/flag seams for prompt-layer roots
- add explicit `--skill <path>` support
- add `--no-skills` support
- optionally add explicit system prompt override flags if needed by the chosen design
- keep merge order aligned with current config policy

## Non-goals

- prompt templates
- package manifests
- interactive settings UI

## Design Notes

- flags should feed typed config, not special-case app branches
- explicit paths should be additive and testable
- keep first pass small: enough to use repo skills, global skills, and ad hoc paths

## Acceptance Criteria

- user can add explicit skill paths from CLI/config
- user can disable discovery while keeping explicit paths possible
- precedence of defaults/file/env/flags remains covered by tests
- docs reflect the new control surface
