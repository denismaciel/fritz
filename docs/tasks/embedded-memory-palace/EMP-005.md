# EMP-005: Gemini embedder adapter

**Type:** Task

## Goal

Implement a Gemini-backed `Embedder` adapter with explicit model config, structured Fritz wiring, and clear failure modes.

## Places To Change

- `pkg/memorypalace/gemini/`
- `internal/memory/`
- `internal/config/`
- `pkg/memorypalace/`

## Scope

- add a Gemini adapter that satisfies `Embedder`
- support model name and endpoint configuration via constructor params, not global env in core logic
- support batch embedding requests
- normalize adapter errors into package-local errors useful to callers
- add tests with a fake HTTP server
- add Fritz-local helper that builds the embedder from structured runtime config

## Non-goals

- DB storage
- vector indexing
- Fritz config loading
- embedding cache beyond what later tickets need

## Design Notes

- keep network IO isolated inside the Gemini adapter package
- constructor should make dependencies explicit
- avoid leaking raw Gemini response structs through exported API
- use the same Gemini api key / endpoint config shape already used elsewhere in Fritz
- make it easy to swap in a fake embedder in tests and later non-Gemini adapters

## Acceptance Criteria

- Gemini adapter satisfies `Embedder`
- tests cover happy path, batch behavior, and non-200/error responses
- Fritz-local helper can construct the adapter from structured runtime config without teaching the core package about env loading

## Dependencies

- `EMP-001`

## Validation

- `go test ./pkg/memorypalace/gemini/... ./internal/memory`
- `rg -n "batchEmbedContents|x-goog-api-key|GeminiEmbedding" pkg/memorypalace/gemini internal/memory internal/config`
