# EMP-006: `sqlite-vec` vector index + hybrid retrieval

**Type:** Task

## Goal

Add `sqlite-vec`-backed vector retrieval and a first hybrid fusion path that combines FTS and vector hits against canonical chunks.

## Places To Change

- `pkg/memorypalace/sqlite/`
- `pkg/memorypalace/`

## Scope

- implement vector index tables and sync paths keyed to canonical chunk IDs
- embed and upsert vectors through the orchestration service without leaking adapter details into callers
- implement a first hybrid search strategy that fuses FTS and vector hits deterministically
- add tests for vector-only, keyword-only, and mixed cases using a fake embedder where practical

## Non-goals

- reranker LLM
- advanced passage-window expansion
- palace-specific closet pointer docs
- approximate-NN tuning beyond what `sqlite-vec` provides out of the box

## Design Notes

- canonical rows remain source of truth; vector index is disposable
- hybrid fusion must be deterministic and simple enough to reason about in tests
- keep reindex operations explicit so model swaps do not silently corrupt search semantics
- support running without vector capability where the caller chooses an FTS-only mode

## Acceptance Criteria

- vector search works when capability is available
- hybrid search can merge FTS and vector results without duplicate chunk hits
- tests cover reindex/update/delete behavior for derived vector rows
- FTS-only mode remains viable when vector capability is absent

## Dependencies

- `EMP-003`
- `EMP-004`
- `EMP-005`

## Validation

- `go test ./pkg/memorypalace/...`
- `rg -n "sqlite-vec|hybrid|Reindex|Search" pkg/memorypalace/sqlite pkg/memorypalace`
