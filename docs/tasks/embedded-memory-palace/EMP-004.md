# EMP-004: FTS5 keyword retrieval adapter

**Type:** Task

## Goal

Add FTS5-backed keyword retrieval over canonical chunks so the first stack has useful search before vector ranking is fully online.

## Places To Change

- `pkg/memorypalace/sqlite/`
- `pkg/memorypalace/`

## Scope

- add derived FTS5 tables or virtual tables for searchable chunk text
- implement a keyword retrieval adapter behind `RetrievalIndex` or a retrieval capability used by the orchestration layer
- support basic metadata scoping needed for source/wing/room-like filters
- add tests for exact-term, filename/name, and filter behavior

## Non-goals

- vector search
- hybrid fusion with vector search
- result reranking beyond deterministic keyword ordering

## Design Notes

- first retrieval quality should favor correctness and explicit filtering over clever ranking
- keep FTS synchronization deterministic with canonical rows
- prefer explicit reindex/update paths over hidden side effects

## Acceptance Criteria

- keyword search returns canonical chunk references with stable ordering rules
- filters work in tests
- FTS index stays consistent after insert/update/delete paths covered by tests

## Dependencies

- `EMP-003`

## Validation

- `go test ./pkg/memorypalace/...`
- `rg -n "FTS5|MATCH|Search" pkg/memorypalace/sqlite pkg/memorypalace`
