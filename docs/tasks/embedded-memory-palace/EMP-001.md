# EMP-001: core package boundary + interfaces

**Type:** Task

## Goal

Create the promotable core package boundary for the memory-palace stack, with domain types and narrow interfaces for canonical storage, embedding, and retrieval.

## Places To Change

- `pkg/memorypalace/`
- `go.mod`
- `docs/memory-palace-storage-notes.md`

## Scope

- add the initial `pkg/memorypalace` package
- define domain types for sources, chunks, search requests, and search hits
- define narrow interfaces for `MemoryStore`, `Embedder`, and `RetrievalIndex`
- add a small orchestration service that composes the interfaces without binding to Fritz runtime types
- add black-box tests that pin the package surface and basic orchestration behavior

## Non-goals

- SQLite implementation
- Ollama HTTP client
- vector math details
- Fritz runtime wiring

## Design Notes

- keep public types minimal and stable
- do not expose third-party DB or HTTP client types in exported signatures
- do not add env/config loading to the core package
- bias toward package docs and tests that make later promotion straightforward

## Acceptance Criteria

- `pkg/memorypalace` exists with a narrow exported API
- public interfaces are Fritz-agnostic
- orchestration tests compile against interface fakes only
- no package outside the new module needs to import SQLite/Ollama types through the core surface

## Dependencies

- `None.`

## Validation

- `go test ./pkg/memorypalace/...`
- `rg -n "type (MemoryStore|Embedder|RetrievalIndex) interface" pkg/memorypalace`
