# EMP-003: canonical SQLite store + schema

**Type:** Task

## Goal

Implement the canonical SQLite-backed `MemoryStore` with explicit schema, migrations, and CRUD for sources and chunks.

## Places To Change

- `pkg/memorypalace/sqlite/`
- `pkg/memorypalace/`

## Scope

- define schema and migrations for sources, chunks, and core metadata
- implement the SQLite `MemoryStore`
- add deterministic IDs or uniqueness rules where needed for safe upsert/reindex behavior
- support listing chunks by source and basic metadata filters needed by later retrieval tickets
- add migration and store behavior tests

## Non-goals

- FTS indexing
- vector indexing
- Ollama calls
- Fritz prompt/resource wiring

## Design Notes

- keep the canonical schema boring and explicit
- chunk text should remain the source of truth; retrieval indexes are derived
- design for reindexability: canonical rows should support delete/rebuild without semantic loss
- do not over-model future graph/fact tables yet

## Acceptance Criteria

- SQLite adapter satisfies `MemoryStore`
- schema creation and migration path are test-covered
- chunk/source CRUD works deterministically
- re-opening the DB preserves stored rows and metadata

## Dependencies

- `EMP-001`
- `EMP-002`

## Validation

- `go test ./pkg/memorypalace/...`
- `rg -n "CREATE TABLE|migration|PutChunk|GetChunk|ListBySource" pkg/memorypalace/sqlite`
