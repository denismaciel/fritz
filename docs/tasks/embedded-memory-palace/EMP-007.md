# EMP-007: Fritz adapter + durable-memory ingest smoke path

**Type:** Task

## Goal

Wire the new package into Fritz through a thin adapter that indexes existing durable memory files and exposes a smoke-tested in-process path without changing current user-visible memory behavior.

## Places To Change

- `internal/memory/`
- `internal/prompt/`
- `pkg/memorypalace/`
- `docs/memory-palace-storage-notes.md`

## Scope

- add a Fritz-local adapter that reads `MEMORY.md` and `memory/*.md` via existing seams and maps them into the new package
- add a small in-process indexing/search smoke path for repo memory content
- keep current durable-memory loading behavior intact while adding the new path
- add tests proving the adapter can index current memory docs and retrieve them via the new stack

## Non-goals

- full agent-wide auto-ingest
- CLI productization
- UI for memory inspection
- broad prompt rewrites

## Design Notes

- this ticket should prove integration without collapsing the package boundary
- reuse existing durable-memory loader where it helps, but keep translation code narrow
- avoid coupling Fritz prompt/resource code directly to SQLite or Ollama internals

## Acceptance Criteria

- Fritz has one thin adapter path into the new package
- existing memory file loading tests still pass
- new smoke tests prove indexing + retrieval over durable memory docs

## Dependencies

- `EMP-003`
- `EMP-004`
- `EMP-005`
- `EMP-006`

## Validation

- `go test ./internal/memory ./internal/prompt ./pkg/memorypalace/...`
- `rg -n "MEMORY.md|memory/.*\\.md|memorypalace" internal/memory internal/prompt pkg/memorypalace`
