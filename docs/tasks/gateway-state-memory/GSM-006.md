# GSM-006: durable memory prompt wiring + explicit write path

**Type:** Task

## Goal

Wire durable memory into prompt assembly and define the explicit write path we will support in v0.

## Places To Change

- `internal/prompt/`
- `internal/app/`
- `README.md`
- `plan.md`

## Scope

- include durable memory docs in the built system/context prompt in a controlled way
- document exact write path for v0:
  - user/operator edits files directly
  - agent may modify them only through normal file tools
- add tests proving memory docs appear in prompt build
- document separation:
  - transcript vs ops state vs durable memory

## Non-goals

- memory summarization
- memory retrieval/search
- memory-specific tools beyond normal file tools

## Design Notes

- keep `G7` explicit and boring
- do not create a hidden database or sidecar service
- if prompt size becomes an issue later, solve that in future retrieval/compaction work, not here

## Acceptance Criteria

- prompt/runtime includes durable memory docs
- tests cover prompt inclusion
- docs clearly explain how durable memory is written and where it lives

## Dependencies

- `GSM-005`

## Validation

- `go test ./internal/prompt ./internal/app`
- `rg -n "MEMORY.md|durable memory|gateway state" README.md plan.md internal/prompt`
