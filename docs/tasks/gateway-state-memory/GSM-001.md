# GSM-001: state paths + atomic JSON store

**Type:** Task

## Goal

Create one package that owns gateway state paths and versioned atomic JSON read/write helpers.

## Places To Change

- `internal/gateway/`
- `internal/telegram/`
- `internal/config/`

## Scope

- add a path resolver for `.fritz/gateway/`
- define named paths for:
  - `meta.json`
  - `routing/session-map.json`
  - `telegram/offset.json`
  - `telegram/allowlist.json`
  - `telegram/pairing.json`
  - `bindings/current.json`
- add atomic JSON write helper used by gateway state code
- add version fields to shared mutable-store payloads
- add tests for path resolution and atomic store behavior

## Non-goals

- migrating all callers yet
- durable memory files
- heartbeat state

## Design Notes

- this ticket creates primitives only
- prefer one small store helper package over ad hoc helpers per feature
- mirror OpenClaw’s concern-based layout:
  - `/home/denis/tmp/openclaw-EUwHTl/src/config/paths.ts`
  - `/home/denis/tmp/openclaw-EUwHTl/extensions/telegram/src/update-offset-store.ts`
- do not touch transcript JSONL/session manager behavior here

## Acceptance Criteria

- code can resolve stable gateway state paths from cwd/config
- mutable stores can be written atomically with versioned payloads
- tests cover path layout and round-trip JSON store behavior

## Dependencies

- `None.`

## Validation

- `go test ./internal/gateway ./internal/telegram ./internal/config`
- `rg -n "version" internal/gateway internal/telegram`
