# GSM-003: split Telegram runtime stores

**Type:** Task

## Goal

Split Telegram offset, allowlist, and pairing into dedicated versioned stores under the Telegram state dir.

## Places To Change

- `internal/telegram/adapter.go`
- `internal/telegram/`
- `internal/app/app.go`
- `internal/config/config.go`

## Scope

- store offset in `telegram/offset.json`
- store allowlist in `telegram/allowlist.json`
- store pairing state in `telegram/pairing.json`
- add typed payloads with `version`
- preserve current pairing-lite behavior
- add migration from current implicit file shapes if needed

## Non-goals

- richer pairing flows
- multi-account Telegram support
- webhooks

## Design Notes

- use OpenClaw refs for file-per-concern layout, not full feature parity:
  - `/home/denis/tmp/openclaw-EUwHTl/extensions/telegram/src/update-offset-store.ts`
  - `/home/denis/tmp/openclaw-EUwHTl/src/pairing/pairing-store.ts`
- pairing file can stay simple in v0; no need for pending request queues
- keep unauthorized group behavior unchanged

## Acceptance Criteria

- Telegram adapter reads/writes concern-specific files only
- restart preserves offset and authorization state
- pairing path is explicit and versioned

- `GSM-001`
