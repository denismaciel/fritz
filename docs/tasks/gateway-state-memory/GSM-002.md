# GSM-002: split routing/session map store

**Type:** Task

## Goal

Replace the generic gateway `state.json` blob with a dedicated routing/session map store.

## Places To Change

- `internal/gateway/gateway.go`
- `internal/gateway/`

## Scope

- move session-key -> session-path persistence to `routing/session-map.json`
- add typed store payload with `version`
- add migration from legacy `state.json` if present
- keep current routing semantics intact
- add tests for migration and restart persistence

## Non-goals

- Telegram allowlist/pairing
- durable memory
- bindings semantics beyond reserving path names

## Design Notes

- this is the first real consumer of `GSM-001`
- preserve `G2`/`G3` behavior exactly
- if legacy `state.json` exists, read once, migrate, and stop writing it
- keep routing store tiny and inspectable

## Acceptance Criteria

- gateway no longer writes mixed `state.json`
- same-user/session continuity still works across restarts
- legacy `state.json` is migrated cleanly in tests

## Dependencies

- `GSM-001`

## Validation

- `go test ./internal/gateway -v`
- `go test ./internal/app -run 'TestRunTelegramPollOnce|TestRunAgentPersistsAndContinuesSession'`
