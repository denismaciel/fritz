# PTP-004 edit core parity

## Goal

Port Pi edit-tool core semantics: exact replacement, multi-edit, overlap handling, and all-or-nothing application.

## Places To Change

- `internal/tool/tool.go`
- `internal/tool/tool_test.go`
- new helper files under `internal/tool/` if needed

## Scope

- exact-match replace
- duplicate detection
- multi-edit application against original file
- overlap rejection
- no partial apply on failure
- diff metadata if needed for future UX

## Non-goals

- fuzzy normalization
- CRLF/BOM preservation
- legacy arg compatibility
- queued serialization

## Design Notes

- Pi matches edits against original file, not incrementally; copy that
- keep edit planning pure, then write once

## Acceptance Criteria

- local tests cover checklist items below
- failed multi-edit leaves file unchanged
- `go test ./...` stays green

## Dependencies

- `PTP-001`
- `PTP-003`

## Validation

- `go test ./internal/tool ./...`

## Test Checklist

- [x] should replace text in file
- [x] should fail if text not found
- [x] should fail if text appears multiple times
- [x] should replace multiple disjoint regions in one call
- [x] should collapse large unchanged gaps in multi-edit diffs
- [x] should match edits against the original file, not incrementally
- [x] should fail when edits is empty
- [x] should fail when multi-edit regions overlap
- [x] should not partially apply edits when one edit fails
