# PTP-008 grep find ls parity

## Goal

Add Pi's discovery tools `grep`, `find`, `ls`, with the tested baseline behaviors Pi relies on for repo exploration.

## Places To Change

- `internal/tool/tool.go`
- `internal/tool/tool_test.go`
- `internal/app/app.go`
- `internal/app/app_test.go`
- `internal/model/model.go` if registry/result shape changes

## Scope

- implement `grep`
- implement `find`
- implement `ls`
- register them in default tool registry
- follow `.gitignore` semantics where Pi does
- expose filenames/context/hidden-file behavior Pi tests

## Non-goals

- every Pi flag or option
- ripgrep subprocess parity if pure Go suffices
- TUI affordances

## Design Notes

- prefer Go-native implementations if simple; shelling out to `rg`/`find` is okay if behavior stays testable
- keep cwd sandbox/path resolver shared with file tools
- after this ticket, local built-ins should match Pi's full built-in set

## Acceptance Criteria

- local tests cover checklist items below
- default registry exposes `grep`, `find`, `ls`
- `go test ./...` stays green

## Dependencies

- `PTP-001`

## Validation

- `go test ./internal/tool ./internal/app ./...`

- [x] should include filename when searching a single file
- [x] should respect global limit and include context lines
- [x] should include hidden files that are not gitignored
- [x] should respect .gitignore
