# PTP-006 edit compatibility and mutation queue parity

## Goal

Port Pi's compatibility shims and file-mutation serialization around `edit`/`write`.

## Places To Change

- `internal/tool/tool.go`
- `internal/tool/tool_test.go`
- `internal/app/app.go`
- new queue/helper files under `internal/tool/` if needed

## Scope

- legacy `oldText`/`newText` compatibility for public edit input
- prepare/normalize edit args before execution
- shared per-file mutation queue across `edit` and `write`
- symlink-aware queue keys
- preserve parallelism for different files

## Non-goals

- TUI diff rendering
- new tools
- provider changes

## Design Notes

- queue belongs near tool execution boundary, not deep in chat core
- same-file mutations serialize; different-file mutations may proceed in parallel
- do not silently expose legacy fields in public schema

## Acceptance Criteria

- local tests cover checklist items below
- same-file edit/write races serialize
- different files can still run concurrently
- `go test ./...` stays green

## Dependencies

- `PTP-003`
- `PTP-004`

## Validation

- `go test ./internal/tool ./internal/app ./...`

## Test Checklist

- [x] keeps legacy fields out of the public schema
- [x] folds top-level oldText/newText into edits
- [x] appends legacy replacement to existing edits
- [x] passes through valid input unchanged
- [x] passes through non-object input unchanged
- [x] prepared args execute correctly
- [x] serializes operations for the same file
- [x] allows different files to proceed in parallel
- [x] uses the same queue for symlink aliases
- [x] preserves both parallel edits on the same file
- [x] shares the queue between edit and write
