# PTP-005 edit normalization parity

## Goal

Port Pi edit matching behavior for fuzzy normalization, CRLF/LF handling, and BOM preservation.

## Places To Change

- `internal/tool/tool.go`
- `internal/tool/tool_test.go`
- new normalization helpers under `internal/tool/` if needed

## Scope

- trailing whitespace normalization
- quote/punctuation/dash/space normalization
- Unicode compatibility normalization
- exact-over-fuzzy preference
- CRLF/LF matching + preservation
- UTF-8 BOM preservation

## Non-goals

- legacy args
- mutation queue
- bash/read/discovery tools

## Design Notes

- exact matches must win before fuzzy matches
- normalization must help matching, not hide duplicates
- preserve original file encoding/line-ending style after edits

## Acceptance Criteria

- local tests cover checklist items below
- CRLF/BOM preserved after successful edits
- duplicates still error after normalization
- `go test ./...` stays green

## Dependencies

- `PTP-004`

## Validation

- `go test ./internal/tool ./...`
