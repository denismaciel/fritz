# SNG-002: add Slack Web API + Socket Mode transport clients

**Type:** Task

## Goal

Implement the low-level Slack clients needed for native Fritz Slack connectivity without bringing in Bolt.

## Places To Change

- `internal/adapters/slack/`
- `internal/logx/`

## Scope

- add typed Slack Web API client methods for:
  - `apps.connections.open`
  - `chat.postMessage`
  - `assistant.threads.setStatus`
  - `assistant.threads.setTitle`
  - `assistant.threads.setSuggestedPrompts`
  - `conversations.replies`
  - `files.getUploadURLExternal`
  - `files.completeUploadExternal`
- add Socket Mode websocket connect/ack/reconnect handling
- decode envelopes and expose typed inbound payloads to the adapter layer
- add focused tests with fake HTTP and websocket servers

## Non-goals

- routing into ingress
- Slack event dedupe
- engine invocation
- streamed message rendering

## Design Notes

- keep Slack API method wrappers narrow and typed; avoid a generic dynamic map client
- ack Socket Mode envelopes immediately after decode, before downstream work
- reconnect behavior should be explicit and testable, not hidden inside a goroutine maze

## Acceptance Criteria

- fake Socket Mode tests show envelope receipt and ack behavior
- reconnect path is covered by tests
- Web API client methods encode expected requests and parse error responses cleanly

## Dependencies

- `SNG-001`

## Validation

- `go test ./internal/adapters/slack -run Test`
- `rg -n "apps.connections.open|chat.postMessage|setStatus|setTitle|setSuggestedPrompts" internal/adapters/slack`
