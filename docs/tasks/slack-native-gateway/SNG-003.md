# SNG-003: normalize inbound Slack events and persist routing state

**Type:** Task

## Goal

Map Slack events into stable Fritz gateway requests with durable dedupe and routing state.

## Places To Change

- `internal/adapters/slack/`
- `internal/ingress/`
- `internal/config/`

## Scope

- normalize `message.im` and `app_mention` into `ingress.InboundMessage`
- derive stable Slack session keys for DMs and thread replies
- extend gateway state paths for Slack state and event dedupe
- persist enough state to avoid duplicate replies on replayed events or reconnects
- add tests for route-key stability and dedupe behavior

## Non-goals

- assistant thread events
- `/clear`
- engine streaming
- artifact upload

## Design Notes

- Slack thread identity should use the root thread timestamp, not transient reply timestamps
- transport-specific state should live under gateway state roots, similar to Telegram
- dedupe should be keyed to Slack event identity, not message text

## Acceptance Criteria

- repeated delivery of the same Slack event does not trigger duplicate handling
- DM events route to a stable per-chat session key
- thread mentions route to a stable per-thread session key
- tests cover normalization and routing edge cases

## Dependencies

- `SNG-002`

## Validation

- `go test ./internal/adapters/slack ./internal/ingress -run 'Test.*Slack|Test.*Route|Test.*Dedupe'`
- `rg -n "slack" internal/ingress internal/adapters/slack`
