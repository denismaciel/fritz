# SNG-004: ship DM and thread mention runs on native Slack transport

**Type:** Task

## Goal

Drive normal Fritz runs from Slack DMs and thread mentions using the existing ingress and engine layers.

## Places To Change

- `internal/adapters/slack/`
- `internal/ingress/`
- `internal/app/`
- `internal/session/`

## Scope

- wire Slack adapter events into gateway `HandleInbound`
- post final replies back to Slack for DM and `app_mention` flows
- persist session paths so repeated Slack conversations resume correctly
- support fetching prior thread messages when needed to preserve thread-local context
- add integration-style tests with fake Slack endpoints

## Non-goals

- assistant thread APIs
- streaming text
- artifact upload

## Design Notes

- reuse the existing ingress service boundary instead of making Slack own session logic directly
- where Slack thread history is fetched, keep it as adapter-level context gathering, not engine redesign
- maintain a clean fallback path to final `chat.postMessage` replies before streaming lands

## Acceptance Criteria

- DM message triggers a Fritz run and returns a Slack reply
- repeated DM messages resume the same Fritz session
- `app_mention` inside a thread replies back into the same thread
- tests cover restart/resume behavior with persisted session paths

## Dependencies

- `SNG-003`

## Validation

- `go test ./internal/adapters/slack ./internal/app ./internal/ingress -run 'Test.*Slack|Test.*DM|Test.*Mention|Test.*Resume'`
- `rg -n "HandleInbound|conversations.replies|postMessage" internal/adapters/slack internal/app`
