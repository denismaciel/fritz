# SNG-001: add Slack config, command wiring, and nix package skeleton

**Type:** Task

## Goal

Create the minimum Fritz surface needed to build and run a native Slack gateway process.

## Places To Change

- `internal/config/`
- `internal/command/`
- `internal/app/`
- `cmd/fritz-slack/`
- `flake.nix`
- `package.nix`

## Scope

- add runtime config for Slack bot token, app token, API endpoints, and basic policy toggles
- add command parsing or dedicated process entrypoint for `fritz-slack`
- add `runSlack(...)` plumbing in the app layer
- package `fritz-slack` as a Nix package and app alongside existing binaries
- keep the binary starting cleanly even before transport implementation lands

## Non-goals

- live Slack websocket handling
- event normalization
- assistant behavior
- message streaming

## Design Notes

- follow the existing `fritz-telegram` process shape rather than inventing a second process model
- keep config validation explicit so missing Slack secrets fail early and clearly
- package changes must not disturb existing `fritz` and `fritz-telegram` outputs

## Acceptance Criteria

- `fritz-slack --help` or equivalent process help works
- runtime config can validate Slack-specific required fields
- `nix build .#fritz-slack` succeeds
- existing `fritz` and `fritz-telegram` package outputs still build

## Dependencies

- `None.`

## Validation

- `go test ./internal/config ./internal/command ./internal/app -run Test`
- `nix build .#fritz-slack`
- `nix build .#fritz .#fritz-telegram`
