# SNG-007: expose streamed engine events to Slack responses

**Type:** Task

## Goal

Refactor the Slack run path to consume Fritz engine events directly and render streamed Slack responses instead of only final text.

## Places To Change

- `internal/ingress/`
- `internal/engine/`
- `internal/adapters/slack/`
- `internal/app/`

## Scope

- add a streamed ingress execution path or event sink abstraction
- map text deltas to `chat.startStream`, `chat.appendStream`, and `chat.stopStream`
- keep a plain `chat.postMessage` fallback for failures or unsupported flows
- optionally map coarse progress/status from engine step events
- add tests for stream lifecycle and fallback behavior

## Non-goals

- redesigning engine event types
- rich Slack task cards for every tool event
- changes to non-Slack transports unless strictly needed by a shared seam

## Design Notes

- Slack should consume engine events without forcing transport details into core runtime packages
- preserve current Telegram/final-text behavior while introducing a richer path for Slack
- fallback path must be explicit so Slack regressions do not strand user responses

## Acceptance Criteria

- Slack responses can stream incremental text from engine deltas
- stream closes cleanly on success and error paths
- fallback final-message path works if streaming APIs fail
- tests cover start/append/stop and fallback behavior

## Dependencies

- `SNG-006`

## Validation

- `go test ./internal/adapters/slack ./internal/ingress -run 'Test.*Stream|Test.*Fallback'`
- `rg -n "startStream|appendStream|stopStream|TextDelta" internal/adapters/slack internal/ingress internal/engine`
