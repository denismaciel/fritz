# SNG-005: add native `/clear` and explicit session reset semantics

**Type:** Task

## Goal

Add a native Slack-side reset command so users can force a fresh Fritz session for the current conversation.

## Places To Change

- `internal/adapters/slack/`
- `internal/ingress/`
- `internal/session/`

## Scope

- detect `/clear` in DM and threaded Slack contexts
- clear the current route's session binding and any route-local cached context
- return a concise user-facing confirmation
- add tests proving isolation between cleared and untouched routes

## Non-goals

- slash command registration
- workspace-global session deletion
- artifact deletion beyond the current route's scoped data

## Design Notes

- reset semantics should be route-local and explicit; do not delete unrelated sessions
- keep the command transport-owned so future channels can implement similar resets cleanly
- this must not depend on assistant-only APIs

## Acceptance Criteria

- sending `/clear` starts a fresh Fritz session for the current Slack route
- unrelated routes keep their session bindings
- tests cover DM and threaded reset paths

## Dependencies

- `SNG-004`

## Validation

- `go test ./internal/adapters/slack ./internal/ingress ./internal/session -run 'Test.*Clear|Test.*Reset'`
- `rg -n "/clear|Reset|clear" internal/adapters/slack internal/ingress internal/session`
