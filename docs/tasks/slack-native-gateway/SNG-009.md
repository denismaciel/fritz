# SNG-009: harden, document, and validate native Slack deployment

**Type:** Task

## Goal

Finish the native Slack transport with docs, regression coverage, and deploy-ready validation.

## Places To Change

- `README.md`
- `docs/tasks/README.md`
- `internal/adapters/slack/`
- `internal/app/`
- `flake.nix`
- optional operator docs under `docs/`

## Scope

- add setup docs for Slack app scopes, events, Socket Mode, and local run commands
- document required secrets and expected state directories
- add regression tests for restart/resume and rate-limit/reconnect paths
- verify `nix build` outputs and a clean end-to-end local run path
- update task-pack index to list the new pack

## Non-goals

- Marketplace-ready HTTP ingress
- feedback buttons or extra Slack product features outside the core pack
- deleting old Bill Nick repos or deployments

## Design Notes

- docs should describe Fritz as the native Slack runtime and Bill Nick as legacy/reference only
- hardening should target the real failure modes: retries, reconnects, duplicate delivery, stale session bindings
- keep operator instructions short and command-focused

## Acceptance Criteria

- repo docs explain how to build and run `fritz-slack`
- test coverage exists for reconnect/retry/resume paths that previously caused operational pain
- `go test ./...` passes
- `nix build .#fritz-slack .#fritz .#fritz-telegram` passes
