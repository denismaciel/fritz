# SNG-008: add run-scoped artifact upload and Slack heartbeat sender

**Type:** Task

## Goal

Deliver generated files and scheduled outbound messages to Slack without reintroducing Bill-style artifact leakage.

## Places To Change

- `internal/adapters/slack/`
- `internal/heartbeat/`
- `internal/ingress/`
- `internal/app/`

## Scope

- define a run-scoped artifact output contract for Slack runs
- upload files with external upload APIs and attach them to the correct Slack thread
- add a Slack heartbeat sender so reminders/wakes can deliver messages natively
- add tests for artifact upload flow and heartbeat send flow

## Non-goals

- conversation-wide artifact scanning
- generalized artifact UI beyond file upload
- redesigning reminder semantics

## Design Notes

- artifacts must be scoped to one run or explicit manifest, not to an entire conversation directory
- heartbeat sender should mirror Telegram sender shape where practical
- keep artifact publication transport-owned; engine should only declare outputs, not speak Slack APIs

## Acceptance Criteria

- one run uploads only its own generated files
- old files are not re-uploaded on later runs unless explicitly regenerated
- heartbeat/reminder code can send a Slack message to a mapped target
- tests cover upload ticketing and completion flow

## Dependencies

- `SNG-007`

## Validation

- `go test ./internal/adapters/slack ./internal/heartbeat -run 'Test.*Artifact|Test.*Upload|Test.*Heartbeat|Test.*Sender'`
- `rg -n "getUploadURLExternal|completeUploadExternal|heartbeat" internal/adapters/slack internal/heartbeat`
