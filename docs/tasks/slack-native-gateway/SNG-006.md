# SNG-006: add assistant thread flows and context-aware Slack APIs

**Type:** Task

## Goal

Support Slack assistant thread entrypoints and assistant-specific APIs as a native Fritz channel.

## Places To Change

- `internal/adapters/slack/`
- `internal/ingress/`
- `internal/app/`

## Scope

- handle `assistant_thread_started`
- handle `assistant_thread_context_changed`
- set suggested prompts on new assistant threads
- set assistant title and loading status for user prompts
- persist assistant thread context for later requests
- add tests for assistant event handling and API method calls

## Non-goals

- token streaming
- feedback buttons
- rich task-update UI

## Design Notes

- assistant thread identity should remain route-compatible with ingress session keys
- context changes should be stored as route metadata, not injected into unrelated sessions
- title and status updates should happen early and clear reliably on completion/failure

## Acceptance Criteria

- assistant thread start can send suggested prompts
- assistant user prompt sets status and title
- context-changed events persist/update assistant context without triggering duplicate replies
- tests cover assistant event normalization and side effects

## Dependencies

- `SNG-004`

## Validation

- `go test ./internal/adapters/slack ./internal/app -run 'Test.*Assistant|Test.*Suggested|Test.*Status|Test.*Title'`
- `rg -n "assistant_thread_started|assistant_thread_context_changed|setSuggestedPrompts|setStatus|setTitle" internal/adapters/slack`
