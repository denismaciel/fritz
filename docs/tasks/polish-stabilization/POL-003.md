# POL-003: packaging and install path

**Type:** Task

## Goal

Make the project easy to build and install locally with a stable binary path and concise commands.

## Places To Change

- `Makefile`
- `README.md`
- `cmd/fritz/`

## Scope

- add or tighten install target(s)
- ensure binary naming/output path is stable
- optionally add version injection if it fits cleanly
- document build/install/run path

## Non-goals

- release automation
- container packaging
- nix packaging changes outside this repo

## Design Notes

- keep install story minimal and local-first
- avoid adding release machinery under polish
- if version injection lands, keep it optional and deterministic

## Acceptance Criteria

- a user can build and install with one short documented command
- README reflects the actual path/targets
- build/install commands are exercised in validation

## Dependencies

- `None.`

## Validation

- `make build`
- `make help`
- `rg -n "install|build|bin/" README.md Makefile`
