# POL-002: bounded Gemini retry policy

**Type:** Task

## Goal

Add narrow, explicit retries for transient Gemini failures without introducing opaque retry behavior.

## Places To Change

- `internal/gemini/`
- `internal/app/`
- `internal/config/`

## Scope

- retry only transient cases like transport failures, `429`, and `5xx`
- bound attempts and delay
- make retry policy configurable if a small config seam is needed
- keep non-retryable failures immediate

## Non-goals

- retries for tool execution
- retries for parse/validation failures
- generalized resilience framework

## Design Notes

- prefer one retry seam in Gemini client or a thin wrapper, not retries scattered across app
- delay/backoff should stay small and deterministic in tests
- surface final error clearly after retries exhaust

## Acceptance Criteria

- transient failures retry and recover in tests
- permanent failures do not retry
- retry behavior is bounded and documented

## Dependencies

- `POL-001`

## Validation

- `go test ./internal/gemini ./internal/app`
- `rg -n "retry|429|5..|backoff" internal`
