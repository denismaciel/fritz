# POL-001: user-facing error surface

**Type:** Task

## Goal

Make user-facing failures clear and categorized instead of leaking raw low-level errors where avoidable.

## Places To Change

- `internal/app/`
- `internal/config/`
- `internal/gemini/`
- `internal/tool/`
- `cmd/fritz/`

## Scope

- define a small error classification/wrapping pattern
- improve messages for config, command parse, session, tool, and Gemini failures
- preserve underlying cause for tests/debugging
- keep stderr/plain output concise

## Non-goals

- logging framework
- telemetry
- deep error UI formatting

## Design Notes

- do not over-abstract; one small shared pattern is enough
- preserve sentinel-style checks where current code relies on them
- prioritize actionable text over stack-ish detail

## Acceptance Criteria

- common failure modes read clearly from CLI
- tests cover a few representative wrapped errors
- no behavior regressions in current command flows

## Dependencies

- `None.`

## Validation

- `go test ./internal/app ./internal/config ./internal/gemini`
- `rg -n "fmt\\.Errorf|errors\\.Is|missing|failed" internal`
