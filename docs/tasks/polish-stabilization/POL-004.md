# POL-004: CLI, help, and docs consistency

**Type:** Task

## Goal

Make command help, README, and current behavior line up exactly.

## Places To Change

- `internal/app/app.go`
- `README.md`
- `plan.md`

## Scope

- tighten help text
- ensure flags/commands/examples match current implementation
- remove stale wording from scaffold-era docs
- add concise examples for sessions, config, and skills where useful

## Non-goals

- long-form docs site
- new commands for discoverability alone

## Design Notes

- prefer short, high-signal docs
- docs should reflect actual behavior only
- keep help output compact; avoid turning it into a manual

## Acceptance Criteria

- help text matches implemented flags/commands
- README examples work as written
- no known doc drift for shipped behavior

## Dependencies

- `POL-003`

## Validation

- `go run ./cmd/fritz help`
- `go run ./cmd/fritz doctor`
- `rg -n "chat|run|doctor|--skill|--continue" README.md internal/app/app.go`
