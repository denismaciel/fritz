# EGT-002: define terminal view-model for text, reasoning, tool calls

**Type:** Task

## Goal

Define a UI-neutral terminal view-model layer that consumes agent events and produces renderable transcript items.

## Places To Change

- `internal/terminalui/` or equivalent new package
- `internal/agent/`
- tests under `internal/terminalui/`

## Scope

- add typed transcript items for:
  - user message
  - assistant text
  - reasoning block
  - tool call/result
  - status/error rows
- support incremental updates from streaming events
- include reduced previews for large tool output
- include collapsed/expanded state hooks even if first cut is mostly collapsed

## Non-goals

- actual Bubble Tea rendering
- keybindings beyond what state needs
- remote HTTP client behavior

## Design Notes

- Bubble Tea model should not parse raw agent events directly
- tool previews should be deterministic and small
- reasoning should have its own item type, not be mixed into assistant text
- keep truncation policy local and testable

## Acceptance Criteria

- tests cover text stream, reasoning stream, tool stream, error stream
- view-model can be driven without terminal IO
- large tool results produce compact previews

## Dependencies

- `EGT-001`

## Validation

- `go test ./...`
- `rg -n "TranscriptItem|ReasoningItem|ToolItem|Preview" internal/terminalui`
