# EGT-001: surface reasoning in internal runtime events

**Type:** Task

## Goal

Add explicit reasoning events to the internal runtime so the embedded terminal UI can render model thinking separately from assistant text.

## Places To Change

- `internal/model/`
- `internal/gemini/`
- `internal/agent/`
- `internal/protocol/agui/`
- `internal/protocol/aisdk/`

## Scope

- extend internal stream/event model for reasoning start/delta/end or equivalent typed reasoning blocks
- extract reasoning from provider responses where already available
- keep assistant text and reasoning as separate event types
- map reasoning into existing protocol adapters where practical

## Non-goals

- Bubble Tea UI
- reasoning editing or post-processing
- provider-wide deep reasoning normalization beyond current Gemini path

## Design Notes

- runtime event model is source of truth; transport adapters follow it
- reasoning can be hidden/collapsed in UI later, but must exist in the event stream now
- keep fallback behavior sane when provider does not expose reasoning

## Acceptance Criteria

- internal runtime emits reasoning events in tests
- plain text and reasoning stay distinct
- AG-UI / AI SDK adapters keep working after event model expansion

## Dependencies

- `None.`

## Validation

- `go test ./...`
- `rg -n "Reasoning" internal/agent internal/model internal/gemini internal/protocol`
