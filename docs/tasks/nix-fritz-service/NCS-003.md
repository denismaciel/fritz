# NCS-003: workspace bootstrap + service hardening

**Type:** Task

## Goal

Make the service start in an empty dedicated workspace with sane defaults and minimal hardening.

## Places To Change

- `../modules/nixos/services/fritz-telegram.nix`

## Scope

- create workspace/home dirs via tmpfiles or equivalent
- seed root workspace files if absent:
  - `AGENTS.md`
  - `MEMORY.md`
  - `HEARTBEAT.md`
  - `memory/`
- set conservative systemd hardening
- set file ownership/permissions for service user

## Non-goals

- host `ben` secret data
- advanced sandboxing that breaks tool/bash behavior
- heartbeat logic changes

## Design Notes

- hardening should be incremental; do not break file tools or bash
- workspace docs are agent-editable; `.fritz/` remains harness-owned state
- seed content should be minimal and operator-readable

## Acceptance Criteria

- unit starts with empty workspace
- required workspace files/dirs exist with right ownership
- hardening opts are present and service still evaluable

## Dependencies

- `NCS-002`

## Validation

- `rg -n "tmpfiles|ProtectSystem|ProtectHome|NoNewPrivileges|WorkingDirectory" ../modules/nixos/services/fritz-telegram.nix`
- `cd .. && nix eval .#nixosConfigurations.ben.config.systemd.services.fritz-telegram.serviceConfig.WorkingDirectory`
