# NCS-004: ben host wiring

**Type:** Task

## Goal

Enable the reusable service on `ben` with thin host-specific config only.

## Places To Change

- `../hosts/ben/configuration.nix`

## Scope

- import/enable new service module on `ben`
- point service env file to `sops` template path
- choose home/workspace paths
- keep heartbeat disabled initially
- keep polling-only deployment

## Non-goals

- defining the secrets themselves
- firewall/webhook exposure
- group chat policy changes

## Design Notes

- match existing host style used for `opencode`
- first deploy should optimize for safe DM smoke test
- host config should stay short; mechanics stay in reusable module

## Acceptance Criteria

- `ben` config evaluates with service enabled
- service unit derives env file, user, workspace from host config cleanly

## Dependencies

- `NCS-002`
- `NCS-003`

- `cd .. && nix eval .#nixosConfigurations.ben.config.services.fritzTelegram.enable`
