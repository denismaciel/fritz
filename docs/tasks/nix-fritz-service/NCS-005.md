# NCS-005: sops-nix secrets + env template

**Type:** Task

## Goal

Provide bootstrap secrets to the service on `ben` via `sops-nix` env template.

## Places To Change

- `../hosts/ben/secrets.nix`
- `../hosts/ben/secrets.yaml`
- `../secrets/shared.yaml`

## Scope

- add Telegram bot token secret
- add fritz pairing token secret
- reuse Gemini key from existing shared secret if possible
- create `sops.templates.fritz_env`
- restart service on relevant secret changes

## Non-goals

- dynamic app-level secret storage in `.fritz/secrets.json`
- non-`ben` host rollout
- secret rotation automation

## Design Notes

- follow `opencode_env` pattern in `../hosts/ben/secrets.nix`
- keep Telegram/Gemini bootstrap secrets out of unit file body
- prefer shared secret reuse when value already exists globally

## Acceptance Criteria

- `ben` eval exposes template path
- env template contains Gemini + Telegram + pairing token placeholders
- no plaintext secrets land in repo-tracked Nix files

## Dependencies

- `NCS-004`

## Validation

- `rg -n "fritz_env|telegram_bot_token|fritz_pairing_token|gemini_api_key" ../hosts/ben/secrets.nix ../hosts/ben/secrets.yaml ../secrets/shared.yaml`
- `cd .. && nix eval .#nixosConfigurations.ben.config.sops.templates.fritz_env.path`
