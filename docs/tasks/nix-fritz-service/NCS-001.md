# NCS-001: package derivation + flake exports

**Type:** Task

## Goal

Package `fritz` as a reusable flake output consumable from GitHub or local host config.

## Places To Change

- `../flake.nix`
- `../nix/fritz.nix`
- `./Makefile`

## Scope

- add `nix/fritz.nix` using Go packaging
- export `packages.<system>.fritz`
- export `apps.<system>.fritz` if low-friction
- keep package invocation reusable from downstream flakes

## Non-goals

- service module
- `ben` host wiring
- `sops-nix`

## Design Notes

- follow existing `nix/opencode-ai.nix` / `pkgs/*.nix` style where sensible
- package should build binary from this repo, not shell out to `go build` ad hoc at runtime
- keep package attrs stable and boring

## Acceptance Criteria

- `nix build .#fritz` works from dotfiles root
- package can be referenced via flake outputs without host-specific glue

## Dependencies

- `None.`

## Validation

- `cd .. && nix build .#fritz`
- `cd .. && nix eval .#packages.x86_64-linux.fritz.name`
