# NCS-002: reusable NixOS service module

**Type:** Task

## Goal

Expose a reusable NixOS module to run the Telegram polling service as a dedicated systemd unit.

## Places To Change

- `../modules/nixos/services/fritz-telegram.nix`
- `../flake.nix`

## Scope

- add module options for package, user, group, home, workspace
- add options for environment file and extra env
- add options for heartbeat enable/interval and pairing token path/env wiring
- create dedicated system user/group
- add systemd service running `fritz telegram`
- export/import module in flake-host usage path

## Non-goals

- `ben`-specific enable stanza
- concrete `sops` secret definitions
- webhook mode

## Design Notes

- follow shape of `../modules/nixos/services/openclaw-gateway.nix`
- keep service generic; host policy belongs in host config
- `WorkingDirectory` should be workspace, not repo checkout

## Acceptance Criteria

- module evaluates in NixOS config without host-specific hacks
- service options cover bootstrap needs without hardcoded host values

## Dependencies

- `NCS-001`

## Validation

- `cd .. && nix eval .#nixosConfigurations.ben.config.services.fritzTelegram.enable`
- `rg -n "services\\.fritzTelegram" ../modules/nixos/services/fritz-telegram.nix ../hosts`
