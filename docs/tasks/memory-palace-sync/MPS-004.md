# MPS-004: engine sync/rebuild flow with keyword-first failure handling

**Type:** Task

## Goal

Implement engine-level document sync so one source can be created, updated, left unchanged, or rebuilt from canonical entries without tying success to vector embedding availability.

## Places To Change

- `pkg/memorypalace/engine.go`
- `pkg/memorypalace/engine_test.go`
- `pkg/memorypalace/interfaces.go`
- `pkg/memorypalace/types.go`

## Scope

- add `SyncDocument`
- add per-source rebuild path
- compute content hash and return action result
- split canonical/FTS write success from optional vector failure
- keep existing search flows working

## Non-goals

- filesystem enumeration
- CLI/app command surface
- session-event ingestion

## Design Notes

- canonical rows, chunks, FTS, and sync state should update atomically per source
- vector failure should degrade to keyword-only rather than fail ingest
- unchanged documents should update seen/scanned state without rewriting content
- avoid embedding when content hash is unchanged

## Acceptance Criteria

- engine reports `created`, `updated`, and `unchanged` correctly
- changed content replaces canonical content and retrieval rows for that source
- vector failure leaves keyword-searchable data intact
- engine tests cover create/update/no-op/rebuild paths

- `MPS-002`
- `MPS-003`
