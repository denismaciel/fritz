# MPS-006: filesystem sync service with missing/tombstone handling

**Type:** Task

## Goal

Add a filesystem document sync service that can ingest external files, detect missing sources, and mark them inactive without hard-deleting by default.

## Places To Change

- `internal/memory/syncdocs.go`
- `internal/memory/syncdocs_test.go`
- `internal/memory/palace.go`
- `pkg/memorypalace/engine.go`
- `pkg/memorypalace/sqlite/adapter.go`

## Scope

- add path normalization and source identity rules for filesystem docs
- enumerate files and call `SyncDocument`
- detect previously seen but now missing files and mark them missing/tombstoned
- keep the service usable from future CLI/app wiring

## Non-goals

- live file watching
- rename detection
- hard-delete pruning policy UI

## Design Notes

- v1 treats rename/move as new source
- missing should mean filtered from default search, not destroyed
- file mtime/size may be used as precheck, but hash stays truth
- keep enumeration and per-file sync separate enough to test

## Acceptance Criteria

- service ingests a directory of files into memory
- deleting a previously synced file marks it missing
- restoring a missing file can reactivate it through sync
- tests cover create/update/delete/resync flows on real temp files

## Dependencies

- `MPS-004`
- `MPS-005`
