# MPS-002: SQLite schema + adapter ops for entries, chunk links, and sync state

**Type:** Task

## Goal

Extend the SQLite adapter with canonical entry tables, chunk-entry mapping, and explicit per-source sync state, while keeping current search behavior intact.

## Places To Change

- `pkg/memorypalace/sqlite/adapter.go`
- `pkg/memorypalace/sqlite/adapter_test.go`
- `pkg/memorypalace/sqlite/doc.go`
- `pkg/memorypalace/types.go`
- `pkg/memorypalace/interfaces.go`

## Scope

- add tables for entries, chunk-entry links, and sync state
- add adapter methods for replace/upsert/list/mark-missing flows
- ensure source hard-delete cascades through new tables
- keep existing source/chunk tables working during migration

## Non-goals

- chunk derivation policy
- engine-level sync orchestration
- app-level file scanning

## Design Notes

- preserve table compatibility where practical; rename later only if needed
- keep migrations deterministic and idempotent
- default search semantics should remain unchanged until filtering ticket lands
- store sync metadata separate from canonical source content

## Acceptance Criteria

- adapter migrates DBs with new tables/indexes cleanly
- adapter can replace entries and chunk-entry mappings for a source
- adapter can upsert and query sync rows by external ref
- adapter can mark source missing and hard-delete source-related rows

## Dependencies

- `MPS-001`
