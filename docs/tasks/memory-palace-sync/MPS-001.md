# MPS-001: core sync types + canonical entry model

**Type:** Task

## Goal

Add the core public types and interfaces needed for canonical entry storage and source-sync without changing behavior yet.

## Places To Change

- `pkg/memorypalace/types.go`
- `pkg/memorypalace/interfaces.go`
- `pkg/memorypalace/engine.go`
- `pkg/memorypalace/doc.go`
- `pkg/memorypalace/engine_test.go`

## Scope

- add canonical entry types alongside existing source/chunk types
- add sync-state types for external documents
- add engine-facing method signatures needed by later tickets
- document invariants around source truth, derived chunks, and sync state

## Non-goals

- SQLite schema changes
- chunk derivation logic
- ingest behavior changes
- command or app wiring

## Design Notes

- keep public API Fritz-agnostic
- keep existing search API stable
- prefer additive changes first so later tickets can migrate behavior incrementally
- sync model must distinguish canonical content from retrieval/index state

## Acceptance Criteria

- core package exposes types for entries, chunk-entry links, and source sync state
- interfaces cover later `SyncDocument` / `MarkMissing` / `RebuildSource` flows without leaking SQLite details
- existing package tests still pass after additive API changes

## Dependencies

- `None.`

## Validation

- `rg -n "type (Entry|SourceSync|SyncSource|SyncResult)|SyncDocument|MarkMissing|RebuildSource" pkg/memorypalace`
- `nix develop -c sh -lc 'export GOFLAGS="-tags=sqlite_fts5"; go test ./pkg/memorypalace/...'`
