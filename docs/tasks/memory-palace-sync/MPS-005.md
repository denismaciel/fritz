# MPS-005: migrate durable-memory ingest to entry-first sync path

**Type:** Task

## Goal

Move the current durable-memory document indexing path onto the new canonical entry + sync flow without changing user-visible behavior.

## Places To Change

- `internal/memory/palace.go`
- `internal/memory/palace_test.go`
- `internal/memory/memory.go`
- `pkg/memorypalace/engine.go`

## Scope

- change durable-memory ingest to build canonical source + entries
- route document writes through `SyncDocument`
- preserve existing search behavior for `MEMORY.md` and `memory/*.md`
- update tests to assert entry-first behavior indirectly through outcomes

## Non-goals

- file scanning for arbitrary external docs
- session-event persistence
- new user-facing commands

## Design Notes

- keep `internal/memory` as thin Fritz wiring over the package
- preserve stable source ids for durable-memory docs
- do not widen Fritz-specific types into the core package

## Acceptance Criteria

- durable-memory indexing still finds the same documents
- repeated indexing of unchanged docs is a no-op at content level
- changed durable-memory docs update search results correctly
- internal memory tests cover unchanged and changed sync cases

## Dependencies

- `MPS-004`

## Validation

- `rg -n "SyncDocument|IndexDocuments|stableID" internal/memory/palace.go internal/memory/palace_test.go`
- `nix develop -c sh -lc 'export GOFLAGS="-tags=sqlite_fts5"; go test ./internal/memory ./pkg/memorypalace/...'`
