# MPS-003: derivation layer for document entries -> retrieval chunks

**Type:** Task

## Goal

Add a deterministic derivation layer that turns canonical document entries into retrieval chunks plus exact chunk-entry mappings.

## Places To Change

- `pkg/memorypalace/derive/`
- `pkg/memorypalace/types.go`
- `pkg/memorypalace/engine_test.go`

## Scope

- create a small derive package or equivalent module
- render one `document_body` entry into one or more stable retrieval chunks
- emit chunk metadata and chunk-entry link rows
- cover paragraph/window chunking for document text

## Non-goals

- session-event chunking
- filesystem sync orchestration
- search filtering

## Design Notes

- chunking must be deterministic for identical content
- chunk metadata should carry derivation version for future policy changes
- keep derivation pure and easily testable
- start with documents only; session grouping comes later

## Acceptance Criteria

- derivation code can take canonical entries and produce chunks plus mappings
- identical input yields identical chunk boundaries and ids
- long document content is split predictably; short content stays simple
- unit tests pin the chunking behavior

## Dependencies

- `MPS-001`
- `MPS-002`

## Validation

- `rg -n "package derive|Derive|chunk.*entry|document_body" pkg/memorypalace`
- `nix develop -c sh -lc 'export GOFLAGS="-tags=sqlite_fts5"; go test ./pkg/memorypalace/... -run Test.*Deriv'`
