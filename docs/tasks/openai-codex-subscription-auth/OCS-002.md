# OCS-002: auth store + locked credential persistence

**Type:** Task

## Goal

Introduce a local auth store for API keys and OAuth creds with file locking and strict separation from session/history state.

## Places To Change

- `internal/auth/` or `internal/authstore/`
- `internal/gateway/paths.go`
- `internal/app/`
- tests under the new package

## Scope

- define global auth path (`$XDG_DATA_HOME/fritz/auth.json`, fallback `~/.local/share/fritz/auth.json`) and typed file format
- support credential kinds:
  - API key
  - OAuth `{access, refresh, expires, accountId}`
- implement file-backed storage with lock-protected read/write/update
- add in-memory backend for tests
- keep tokens out of logs and stringers
- keep refresh-token-bearing types confined to this package

## Non-goals

- provider-specific OAuth logic
- command UX
- gateway request signing

## Design Notes

- mirror Pi’s locking principle, not necessarily its exact type graph
- writes should preserve `0600`-style permissions
- this package should be reusable by future providers, not Codex-only
- borrow path/atomic-write style from local secret storage if useful, but add real lock-safe update API

## Acceptance Criteria

- auth store can save, load, update, and delete credentials
- concurrent refresh/update path is lock-safe in tests
- auth file path is stable and not mixed with session files
- downstream packages cannot access refresh token unless explicitly using authstore APIs

## Dependencies

- `OCS-001`

## Validation

- `go test ./internal/...`
- `rg -n "auth.json|refresh|lock" internal`
