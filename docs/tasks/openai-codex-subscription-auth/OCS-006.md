# OCS-006: runtime wiring + provider selection

**Type:** Task

## Goal

Wire `openai-codex` into the normal run/chat/serve flow so the main agent can actually execute through the new provider.

## Places To Change

- `internal/app/`
- `internal/agent/`
- `internal/httpapi/`
- `internal/telegram/` if provider-sensitive doctor/status data is exposed

## Scope

- ensure selected provider flows through run/chat/serve/bootstrap paths
- make doctor/status reflect selected provider and auth state
- verify session/compaction paths still work with the new gateway
- add end-to-end-ish tests with fake Codex server/gateway
- keep provider branching in app/bootstrap only

## Non-goals

- websocket support
- advanced per-model registry UX
- multi-provider automatic failover

## Design Notes

- do not fork agent logic by provider; keep divergence in gateway/auth edges
- compaction, tools, and session code should stay provider-agnostic
- if provider-specific quirks surface, isolate them in request/response adapter code
- no registry creep: a small provider switch is enough for v1

## Acceptance Criteria

- operator can select `openai-codex` and run the main agent flow
- doctor/status output makes auth/provider state visible
- existing Gemini tests stay green

## Dependencies

- `OCS-005`

## Validation

- `go test ./...`
- `rg -n "openai-codex|provider:" internal/app internal/agent internal/httpapi`
