# OCS-007: refresh hardening + docs + operator checks

**Type:** Task

## Goal

Harden token refresh/error handling and document the operator flow so subscription auth is usable without tribal knowledge.

## Places To Change

- auth/OAuth packages from prior tickets
- `internal/app/`
- `docs/`
- tests around refresh and friendly failures

## Scope

- refresh expired OAuth token before request execution
- lock refresh path so concurrent runs do not corrupt auth state
- surface friendly operator errors for expired/invalid subscription auth
- add docs for login, logout, status, and expected local files
- add `doctor` or equivalent checks for stored Codex auth
- keep refresh orchestration in auth layer; request code should only consume resolved request auth

## Non-goals

- websocket support
- GUI/browser auth UX
- supporting every OpenAI account edge case

## Design Notes

- mirror Pi’s refresh-lock idea:
  - `/tmp/pi-mono/packages/coding-agent/src/core/auth-storage.ts`
- keep refresh isolated and testable
- docs should call out maintenance risk and subscription/backend dependency explicitly
- refresh path should expose a narrow interface to app/bootstrap, not bleed into transport code

## Acceptance Criteria

- expired stored token is refreshed automatically before request use
- concurrent refresh path is safe in tests
- docs explain the operator flow end-to-end
- full repo tests pass

- `OCS-006`
