# OCS-005: codex sse gateway client

**Type:** Task

## Goal

Implement an `openai-codex` gateway client that talks to ChatGPT Codex backend over SSE using subscription OAuth auth.

## Places To Change

- new `internal/openaicodex/` or `internal/provider/openai/`
- `internal/model/`
- `internal/app/`
- focused tests for request/response mapping

## Scope

- POST to `https://chatgpt.com/backend-api/codex/responses`
- send required headers:
  - `Authorization: Bearer <access>`
  - `chatgpt-account-id: <accountId>`
  - `OpenAI-Beta: responses=experimental`
  - `originator: fritz` or chosen local originator string
- map local request/messages/tools into Codex request body
- decode SSE response stream into local `model.Response`
- support non-stream fallback if needed for shared interface compatibility
- add tests for header shape and SSE parsing
- consume request-ready auth only; never read auth store or refresh tokens here

## Non-goals

- websocket transport
- remote prompt downloads
- every Codex-specific advanced field on day one

## Design Notes

- Pi reference transport:
  - `/tmp/pi-mono/packages/ai/src/providers/openai-codex-responses.ts`
- keep this client isolated from Gemini code
- match the minimal proven header contract first; expand only if required by tests/manual checks
- treat this package as transport adapter only, not auth orchestrator

## Acceptance Criteria

- local gateway can send a Codex SSE request with correct auth headers
- stream parsing yields local assistant text/tool-call structures
- focused transport tests cover request header contract and basic event mapping
- transport tests do not require auth-store knowledge
