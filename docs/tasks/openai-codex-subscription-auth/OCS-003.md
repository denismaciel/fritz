# OCS-003: codex oauth login/logout command flow

**Type:** Task

## Goal

Add CLI/operator flow to log in to ChatGPT subscription-backed `openai-codex`, persist OAuth credentials, and log out cleanly.

## Places To Change

- `internal/command/`
- `internal/app/`
- new provider-specific OAuth package under `internal/openaicodex/` or `internal/auth/`
- docs under `docs/`

## Scope

- implement PKCE auth flow against:
  - `https://auth.openai.com/oauth/authorize`
  - `https://auth.openai.com/oauth/token`
- use localhost callback server with manual paste fallback
- persist `access`, `refresh`, `expires`, `accountId`
- add CLI commands:
  - `auth login openai-codex`
  - `auth logout openai-codex`
  - optional `auth status`
- add focused tests for callback parsing, state validation, and token decode/accountId extraction

## Non-goals

- request transport to Codex backend
- automatic refresh during requests
- browser-only UI integration

## Design Notes

- use Pi’s source flow as reference:
  - `/tmp/pi-mono/packages/ai/src/utils/oauth/openai-codex.ts`
- keep provider-specific constants isolated
- do not print tokens to stdout/stderr
- login flow should fail closed on state mismatch or missing account id
- keep network/browser/process orchestration at shell edge; parsing/validation helpers should stay pure

## Acceptance Criteria

- operator can log in and persist Codex OAuth credentials
- operator can remove stored Codex credentials
- tests cover the critical OAuth parsing/validation logic

- `OCS-002`
