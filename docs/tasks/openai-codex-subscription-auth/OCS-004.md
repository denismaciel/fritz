# OCS-004: request-auth resolver + bootstrap seam

**Type:** Task

## Goal

Resolve request auth from the new auth store and make bootstrap use provider-selected credentials instead of raw env-only Gemini config.

## Places To Change

- `internal/app/`
- `internal/config/`
- new `internal/auth/` or `internal/provider/` resolver package

## Scope

- add provider auth resolution seam used by app bootstrap
- resolve current credential for selected provider from auth store
- for OAuth creds, expose access token + accountId without leaking refresh token downstream
- keep Gemini env-key path working
- add tests for missing auth, present auth, and provider mismatch cases
- define a small request-auth type suitable for gateway construction / request use

## Non-goals

- Codex HTTP client itself
- token refresh network calls
- generalized model registry
- changing `internal/model.Gateway`

## Design Notes

- keep refresh token handling in auth layer, not gateway layer
- gateway constructors should receive only what they need for requests
- failure messages should be operator-usable but token-safe
- prefer a shape like `RequestAuth{BearerToken, Headers}` over provider-specific leakage

## Acceptance Criteria

- app bootstrap can resolve provider auth from local auth store
- missing/invalid auth fails before model requests
- Gemini flow remains functional
- gateway/bootstrap consumers receive request-ready auth, not refresh creds

## Dependencies

- `OCS-001`
- `OCS-002`
- `OCS-003`

## Validation

- `go test ./internal/app ./internal/...`
- `rg -n "accountId|auth store|resolve.*auth|provider switch" internal`
